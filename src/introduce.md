# src 目录结构与功能说明

```

│   ├── model.py               # 模型定义
│   ├── train_eval.py          # 训练与评估
│   ├── continual.py           # 持续学习相关
│   └── fed_continual.py       # 联邦持续学习
├── datasets/                  # 数据集处理
│   ├── __init__.py
│   ├── cifar_svhn.py          # 数据集加载
│   ├── data_loader.py         # 数据流程编排
│   ├── data_split.py          # 数据集分割
│   └── client_dataloader.py   # DataLoader 工具
├── experiments/               # 实验脚本
│   ├── __init__.py
│   ├── cifar10/
│   │   ├── __init__.py
│   │   ├── centralized_cifar10.py # 集中式学习脚本
│   │   ├── fedavg_cifar10.py      # 联邦学习脚本
│   │   ├── continual_cifar10.py   # 持续学习脚本
│   │   └── federated_continual_cifar10.py # 联邦持续学习脚本
│   └── svhn/                   # SVHN 实验（预留）
└── utils/                     # 工具模块
    ├── config.py              # 配置管理
    ├── results_standard.py    # 标准学习结果处理
    └── results_continual.py   # 持续学习结果处理
```

## 核心模块功能

### 顶层文件

| 文件                       | 功能                                                              |
| -------------------------- | ----------------------------------------------------------------- |
| **main.py**          | 项目总入口，支持运行三种实验模式：centralized/federated/continual |
| **config.json**      | 实验配置文件，包含超参数设置                                      |
| **introduce.md**     | 本文档，src 目录结构与功能说明                                    |

### core/ 核心算法模块

| 文件                       | 功能                                           |
| -------------------------- | ---------------------------------------------- |
| **fedavg.py**        | 实现 FedAvg 联邦学习算法，包括全局参数加权平均 |
| **model.py**         | 模型定义模块，包含 SimpleCNN 等网络结构        |
| **train_eval.py**    | 包含训练循环、本地训练、模型评估等基础函数     |
| **continual.py**     | 持续学习相关算法和工具                         |
| **fed_continual.py** | 联邦持续学习算法，结合联邦学习和持续学习       |

### datasets/ 数据处理模块

| 文件                           | 功能                                                      |
| ------------------------------ | --------------------------------------------------------- |
| **cifar_svhn.py**        | 加载原始 CIFAR-10 和 SVHN 数据集，支持划分 train/val/test |
| **data_split.py**        | 实现数据集分割算法，支持 IID 平均切分给多个客户端         |
| **data_loader.py**       | 统一数据准备流程，根据实验类型返回对应的 DataLoader       |
| **client_dataloader.py** | DataLoader 工厂函数，为训练/测试/多客户端构造数据加载器   |

### experiments/ 实验脚本

| 文件                                             | 功能                                                       |
| ------------------------------------------------ | ---------------------------------------------------------- |
| **cifar10/centralized_cifar10.py**         | 集中式学习基线实验，在完整 CIFAR-10 上训练单个模型         |
| **cifar10/fedavg_cifar10.py**              | 联邦学习实验，多客户端本地训练 + 全局聚合                  |
| **cifar10/continual_cifar10.py**           | 持续学习实验，分两个任务阶段，观察灾难性遗忘，支持经验重放 |
| **cifar10/federated_continual_cifar10.py** | 联邦持续学习实验，结合联邦学习和持续学习场景               |

### utils/ 工具模块

| 文件                           | 功能                                        |
| ------------------------------ | ------------------------------------------- |
| **config.py**            | 配置加载、结果目录创建、配置文件保存        |
| **results_standard.py**  | 保存和分析标准学习实验结果（CSV/JSON/图表） |
| **results_continual.py** | 持续学习结果处理，计算遗忘指标并生成可视化  |

## 使用示例

```bash
# 运行所有实验
python -m src.main all

# 运行特定实验
python -m src.main centralized   # 集中式学习
python -m src.main federated     # 联邦学习
python -m src.main continual     # 持续学习
python -m src.main fed_continual # 联邦持续学习
```

## 配置说明

项目使用 `config.json` 进行配置管理，主要参数包括：

- `experiment_name`: 实验名称
- `dataset`: 数据集选择 ("cifar10" 或 "svhn")
- `model`: 模型类型 ("SimpleCNN")
- `num_clients`: 联邦学习客户端数量
- `num_rounds`: 联邦学习总轮数
- `local_epochs`: 每个客户端本地训练轮数
- `batch_size`: 批大小
- `lr`: 学习率
- `weight_decay`: 权重衰减
- `seed`: 随机种子
- `split_type`: 数据分割类型 ("iid")
- `device`: 计算设备 ("cuda" 或 "cpu")

## 实验结果

### 1. 集中式学习基线 (Centralized)
**配置**: 50轮训练，学习率0.001
- **测试准确率**: 68.43%
- **测试损失**: 0.9149
- **验证准确率**: 69.10%
- **训练时间**: 2354.5秒

### 2. 联邦学习 (FedAvg)
**配置**: 3个客户端，50轮通信，每轮3个本地epochs，学习率0.001
- **测试准确率**: 67.14%
- **测试损失**: 1.0580
- **验证准确率**: 67.06%
- **性能对比**: 相比集中式下降1.29个百分点（通信开销与本地更新权衡）

### 3. 持续学习 (Continual Learning)
分两种场景测试，包含3个任务阶段：

#### 3.1 无经验重放 (No Replay)
- **最终遗忘指标**: 81.93% （严重的灾难性遗忘）
- **任务1最终准确率**: 0.00% （完全遗忘）
- **任务2最终准确率**: 0.00% （完全遗忘）
- **任务3最终准确率**: 92.15% （当前任务性能良好）
- **平均准确率**: 30.72% （非常低，说明模型无法保持旧任务知识）

#### 3.2 有经验重放 (With Replay, 20%重放率)
- **最终遗忘指标**: 36.97% （显著改善）
- **任务1最终准确率**: 54.30% （保持一定性能）
- **任务2最终准确率**: 32.87% （部分保持）
- **任务3最终准确率**: 86.28% （略有下降）
- **平均准确率**: 57.81% （大幅改善，提升89%）

**关键发现**: 经验重放有效减缓灾难性遗忘，遗忘指标降低55%，平均准确率提升近两倍。

### 4. 联邦持续学习 (Federated Continual Learning)
结合了联邦学习和持续学习场景，3个客户端，3个任务阶段：

#### 4.1 无经验重放 (No Replay)
- **最终遗忘指标**: 78.77% （严重遗忘）
- **任务1最终准确率**: 0.00% （完全遗忘）
- **任务2最终准确率**: 0.00% （完全遗忘）
- **任务3最终准确率**: 89.88% （当前任务性能优秀）
- **平均准确率**: 29.96% （极低）

#### 4.2 有经验重放 (With Replay, 20%重放率)
- **最终遗忘指标**: 32.68% （显著改善）
- **任务1最终准确率**: 49.97% （保持性能）
- **任务2最终准确率**: 39.97% （部分保持）
- **任务3最终准确率**: 82.80% （下降幅度较小）
- **平均准确率**: 57.58% （大幅改善，提升92%）

**关键发现**: 经验重放同样有效，遗忘指标降低59%，平均准确率提升近两倍。联邦学习与持续学习结合时，重放机制同样关键。

### 结果总结

| 实验类型 | 平均准确率 | 关键指标 | 备注 |
| -------- | --------- | ------- | ---- |
| 集中式学习 | 68.43% | 基线 | 单机训练 |
| 联邦学习 | 67.14% | -1.29% | 3客户端分布式 |
| 持续学习(无重放) | 30.72% | 遗忘81.93% | 严重灾难性遗忘 |
| 持续学习(有重放) | 57.81% | 遗忘36.97% | 经验重放有效 |
| 联邦持续学习(无重放) | 29.96% | 遗忘78.77% | 分布式场景更差 |
| 联邦持续学习(有重放) | 57.58% | 遗忘32.68% | 经验重放缓解 |

### 关键结论

1. **灾难性遗忘的严重性**: 在不采用任何缓解策略的情况下，模型在新任务学习时会完全遗忘旧任务，遗忘指标超过78%。

2. **经验重放的有效性**: 即使只采用20%的数据重放率，也能将遗忘指标从78-82%降低到32-37%，平均准确率提升接近两倍。

3. **联邦学习的通信代价**: 相比集中式学习，联邦学习性能下降1.29%，这是通信频率、本地更新次数和全局聚合机制之间的权衡结果。

4. **联邦+持续学习的挑战**: 当同时面临分布式学习和任务切换的挑战时，模型性能降低更多，但经验重放机制仍然有效。
